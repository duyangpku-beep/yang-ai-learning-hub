# Yang HTML Agent Kit

Two files. Drop them into any agent environment.

---

## Files

| File | Purpose |
|------|---------|
| `html-system-prompt.md` | Inject into agent system prompt → on-brand HTML output |
| `deploy.py` | Deploy any HTML string to Vercel via API → returns shareable URL |

---

## Quick Start

### 1. Get a Vercel token (one-time)

Go to **vercel.com/account/tokens** → Create → copy the token.

Set it as an environment variable in your agent's runtime:
```bash
export VERCEL_TOKEN=your_token_here
```

On Tencent Cloud SCF / container: add as **环境变量** in the function config.

---

### 2. Feed the design system to your agent

Fetch the system prompt at runtime:
```python
import urllib.request
url = "https://raw.githubusercontent.com/duyangpku-beep/yang-ai-learning-hub/main/agent-kit/html-system-prompt.md"
with urllib.request.urlopen(url) as r:
    design_system_prompt = r.read().decode("utf-8")

# Prepend to your agent's system message:
system_message = design_system_prompt + "\n\n" + your_existing_system_prompt
```

Or hardcode the raw URL directly in your system prompt instructions:
```
Before generating any HTML, fetch and follow the design system at:
https://raw.githubusercontent.com/duyangpku-beep/yang-ai-learning-hub/main/agent-kit/html-system-prompt.md
```

---

### 3. Deploy generated HTML

**Option A — command line:**
```bash
# From a file
python3 deploy.py --slug "meeting-report-20260512" --file report.html

# From stdin / string
python3 deploy.py --slug "my-report" --html "<html>...</html>"
```

**Option B — import in Python agent:**
```python
from deploy import deploy_html

html = "..."  # generated by your LLM
url = deploy_html("meeting-report-20260512", html)
print(url)    # → https://yang-meeting-report-20260512.vercel.app
```

**Option C — pure curl (no Python needed):**
```bash
HTML_CONTENT=$(cat report.html | python3 -c "import sys,json; print(json.dumps(sys.stdin.read()))")

curl -s -X POST https://api.vercel.com/v13/deployments \
  -H "Authorization: Bearer $VERCEL_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"yang-my-report\",\"files\":[{\"file\":\"index.html\",\"data\":$HTML_CONTENT}],\"projectSettings\":{\"framework\":null},\"target\":\"production\"}" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print('https://'+d.get('url',''))"
```

## URL Behaviour

| Scenario | URL |
|----------|-----|
| First deploy with slug `abc` | `https://yang-abc-[hash].vercel.app` |
| Same slug deployed again | Same alias — **content updated, URL unchanged** |
| Different slug | New project, new URL |

Use descriptive slugs: `meeting-notes-20260512`, `deal-summary-q2`, `pr-review-feature-x`

## Tencent Cloud SCF Example

```python
import os
import urllib.request
import json

# Load deploy helper (bundle deploy.py with your function)
from deploy import deploy_html

def handler(event, context):
    # 1. Get design system (cache this — don't fetch every invocation)
    ds_url = "https://raw.githubusercontent.com/duyangpku-beep/yang-ai-learning-hub/main/agent-kit/html-system-prompt.md"
    with urllib.request.urlopen(ds_url) as r:
        design_system = r.read().decode("utf-8")

    # 2. Call your LLM with design system in system prompt
    html = call_your_llm(
        system=design_system,
        user=event.get("prompt", "Generate a report")
    )

    # 3. Deploy and return URL
    slug = event.get("slug", "agent-output-" + context.request_id[:8])
    url = deploy_html(slug, html)

    return {"url": url, "message": f"Deployed to {url}"}
```

Environment variable needed: `VERCEL_TOKEN`

## Notes

- `deploy.py` has **zero dependencies** — pure Python stdlib, works on any Python 3.4+
- HTML must be self-contained (inline CSS, no local file references)
- Same project name = content update, URL stays the same (good for recurring reports)
- Free Vercel hobby plan supports unlimited deployments
